The Infamous N+1 Problem

The most common performance killer in WordPress - running queries inside loops

LIVE DEMO 
// BAD: N+1 Queries
foreach ( $posts as $post ) {
    $meta = get_post_meta( $post->ID, 'custom_field', true );
    // This runs a query for EACH post!
}

// GOOD: Single Query
$post_ids = wp_list_pluck( $posts, 'ID' );
update_meta_cache( 'post', $post_ids );
// Now get_post_meta uses cache!

The db.php Magic

Bypass WordPress restrictions with a custom database layer to capture every query and error 

// wp-content/db.php
<?php
define( 'SAVEQUERIES', true );

add_action( 'shutdown', function() {
    global $wpdb;
    $log = fopen( WP_CONTENT_DIR . '/query-log.txt', 'a' );
    
    foreach ( $wpdb->queries as $q ) {
        fwrite( $log, sprintf( 
            "[%s] %fs - %s\n",
            date('H:i:s'), $q[1], $q[0]
        ));
    }
    fclose( $log );
});

require_once ABSPATH . WPINC . '/wp-db.php';

This works even when WordPress is broken!

Smart Caching Strategies

Implement multi-layered caching to dramatically reduce database queries and server load 

// Level 1: Object Cache (per request)
$data = wp_cache_get( 'my_data' );
if ( false === $data ) {
    $data = expensive_operation();
    wp_cache_set( 'my_data', $data );
}

// Level 2: Transients (persistent)
$data = get_transient( 'my_data' );
if ( false === $data ) {
    $data = expensive_operation();
    set_transient( 'my_data', $data, HOUR_IN_SECONDS );
}

// Level 3: Static caching (same request)
static $data = null;
if ( null === $data ) {
    $data = expensive_operation();
}

Use the right cache for the right job!
